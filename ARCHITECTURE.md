# Model Architecture

## Vision Encoder

The Vision Encoder is a single-layer Convolutional 2D Neural Net, we use it only to make tokens/patches from images.

## Transformer

We use a Transformer Encoder ([learn more here](https://www.arxiv.org/abs/1706.03762)) to process the tokens/patches from the CNN and the actions; we also add a single *Class* token that is masked during attention ([learn more here](https://arxiv.org/abs/1810.04805)).

We encode the actions in vectors/tokens by repeting D times (D is the input dimensions of the Transformer) the value of the action.

## Decoder

The Decoder is used only to render latent frames generated by the Transformer to "play" inside the world model. So the Decoder is a simple Multi-Layer Perceptron generating each pixels of each frames.

It's not a smart choice, but DOOM is played in low resolutions so it's fine.

# Where's All the Data?

The problem in data is:
1. We need a lot of data on the first episode of DOOM
2. We need action anotated data

The solution is to a human to play inside VizDoom to record all of it's actions. Then train a model to label actions between two frames, and then to use this model to label internet gameplay of DOOM on the first episode.
